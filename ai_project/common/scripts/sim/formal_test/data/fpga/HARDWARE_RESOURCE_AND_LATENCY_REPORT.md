### 6.1 量化测试结果汇总

### 6.3 输出差异分析

| 指标 | INT8 | INT16 Symmetric | INT16 Asymmetric |
|------|------|-----------------|-----------------|
| Max Absolute Difference | 214.98 | 214.92 | 300.15 |
| Avg Absolute Difference | 15.16 | 15.18 | 29.42 |
| Std Absolute Difference | 28.66 | 28.62 | 46.70 |

### 6.4 量化问题分析

**问题**: INT8和INT16量化后F1分数均大幅下降（>50%）

**可能原因**:
1. 后训练量化（PTQ）未考虑激活值量化，仅量化权重导致误差累积
2. 模型权重范围较大(-3.08~3.16)，量化分辨率不足
3. 输出范围较大(-255~1.6)，量化误差被放大
4. 偏差未量化，与权重量化后的尺度不匹配

**权重归一化效果**:

| 方法 | 原始Max Range | 归一化后Max Range | 效果 |
|------|---------------|-------------------|------|
| Original | 6.24 | - | - |
| layer_norm | 14.15 | ❌扩大 | 不推荐 |
| range_norm | 1.97 | ✅缩小 | **推荐** |
| max_norm | 2.00 | ✅缩小 | **推荐** |
| quantization_aware | 6.24 | ❌无变化 | 不推荐 |

## 9. 部署建议

### 9.1 当前状态总结

| 指标 | 状态 | 说明 |
|------|------|------|
| FPGA资源 (INT16) | ✅ 充足 | DSP48E利用率89.1% |
| FPGA资源 (INT8) | ✅ 充足 | DSP48E利用率44.6% |
| FPGA资源 (FP32) | ❌ 不足 | DSP48E超出可用量78% |
| CPU实时性 | ✅ 满足 | 3.84ms < 100ms |
| FPGA实时性 | ✅ 满足 | 0.6ms < 100ms |
| INT8/INT16量化精度 | ❌ 需优化 | F1损失50%+ |

### 9.2 推荐方案

**短期方案**：使用INT16模型部署到FPGA，接受当前精度损失

**中期方案**：
1. 使用权重归一化（range_norm/max_norm）重新训练模型
2. 对归一化后的模型进行INT8量化
3. 预期可将精度损失控制在10%以内

**长期方案**：探索量化感知训练（QAT）或混合精度方案

## 10. 结论

### 10.1 核心发现

1. **CPU推理延迟**: SAGE2-Lite-64平均3.838ms，完全满足实时性要求
2. **FPGA资源**: INT16方案DSP48E利用率89.1%，INT8方案仅44.6%，均充足
3. **FPGA加速比**: 理论加速6.4x，吞吐量提升38.5x
4. **FP32部署**: 在XC7Z020上不可行，DSP48E资源溢出78%
5. **INT8/INT16量化**: 当前PTQ方案精度损失较大(50%+)，需要进一步优化
6. **权重归一化**: range_norm和max_norm能有效将权重范围限制在[-1,1]，为量化打下良好基础

### 10.2 建议

1. **量化优化**: 使用权重归一化重新训练模型，再进行INT8/INT16量化
2. **部署优先级**: 在XC7Z020上使用INT16方案，平衡精度与资源
3. **进阶方案**: 探索量化感知训练（QAT）提升量化精度
4. **性能监控**: 在实际FPGA板卡上测试，获取真实延迟数据
