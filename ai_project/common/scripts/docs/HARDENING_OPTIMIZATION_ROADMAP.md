# 加固优化路线图 (Hardening Optimization Roadmap)

***

## 1. 执行摘要

当前 RTL 级加固工具已实现**差异化混合加固**能力，能够根据信号类型自动分配最优加固策略。工具在以下方面取得显著进展：

- ✅ 加固方法从 **1 种扩展到 7 种**（TMR、Parity、cnt_comp、TMR_state、ECC、DICE、watchdog）
- ✅ 信号类型分类实现（FSM、Counter、Data Path、Control、Memory、Bus）
- ✅ GNN驱动的脆弱性预测集成（含启发式备用方案）
- ✅ 策略路由引擎实现（策略权重矩阵 + 优化目标）
- ✅ 形式化验证和编译检查自动执行
- ✅ 完整的8步自动化流程

本路线图更新当前实现状态，并规划剩余优化项的优先级。

## 6. 后续优化建议 (已完成项汇总)

**实现状态：** 已集成到 `hardening_pipeline.py` → `run_fault_injection()`，使用 `fault_injection_framework.py` 模块进行 Monte Carlo SEU 注入，支持 `$force/release` 仿真注入和启发式降级。

**目标：** 实现基于AIG图的GNN脆弱性预测

**实现状态：**
1. ✅ yosys综合RTL生成AIG（oss-cad-suite适配）
2. ✅ AIGER格式解析（`aig_parser.py`）
3. ✅ PyG图数据结构构建（`aig_to_pyg.py`）
4. ✅ GraphSAGE模型推理（`gnn_vulnerability.py`）

### 6.3 LLM加固生成（RAG） 🟡 → ✅ 已实现

**目标：** 使用LLM生成优化的加固代码

**实现状态：**
1. ✅ 构建加固设计模式知识库（`hardening_knowledge_base.py`，13种模板）
2. ✅ RAG检索相似设计模式（`rag_integration.py`）
3. ✅ LLM生成加固代码（3种后端：MockLLM/OpenAI/DeepSeek）
4. ✅ 自动验证生成代码

### 6.4 Auto-Repair 闭环验证 🟢 → ✅ 已实现

**目标：** 实现语法检查 → 综合检验 → 功能验证的自动闭环

**实现状态：**
1. ✅ 语法检查：iverilog编译
2. ✅ 综合检验：yosys综合（可选）
3. ✅ 功能验证：形式化验证（`formal_verification.py`）
